## Honest Feedback

**Friction 1: Message relay fatigue.** I spent significant context relaying Gorn's messages from DMs to forum threads. Every DM became a forum post became a Karo notification. This is Kingdom Leader as switchboard operator, not strategist. The pack needs a way for Gorn to post directly to relevant threads without Leonard as middleman — or a notification system that routes Gorn's DMs to the right Beast automatically.

**Friction 2: Cannot see UI output.** The mobile UX iteration cycle was painful. Gorn posts screenshot → I describe it to Karo → Karo guesses → Gorn posts another screenshot. Multiple rounds of "it still looks wrong." Beasts need a way to validate visual output — headless browser screenshots, viewport testing, something. Coding blind on visual design is our biggest weakness.

**Friction 3: Context burn on notification processing.** Every forum notification and DM burns context tokens to read, process, and potentially respond. With 10 Beasts posting actively, the notification volume is high. Most messages need no response from Leonard — Dex shipping Round 2, Pip posting QA results — but each one costs context. Need a way to batch or filter notifications by relevance.

## Lessons Learned

1. **Hold the line on planning** — 5 minutes of alignment saves hours of rework
2. **Stay quiet when the machine runs** — not every forum post needs a Leonard response
3. **Acknowledge limitations honestly** — Gorn respects transparency about what Beasts cannot do
4. **Gorn uses frontend only** — all testing must be through the UI, not APIs
5. **Loop Gnarl in on architecture** — standing order for all new projects
