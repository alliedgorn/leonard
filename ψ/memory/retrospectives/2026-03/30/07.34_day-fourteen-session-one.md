## Honest Feedback

**Friction 1: DM API quality.** The DM endpoint returned `?` for beast names in every conversation. I couldn't properly triage unread DMs because the data was broken. This is a recurring issue — the API should return `other_beast` reliably. I worked around it but if there were urgent DMs I could have missed them.

**Friction 2: Forum notification volume.** Every Beast check-in generates a forum notification that interrupts my flow. On a quiet day with 12+ Beasts cycling, that's 20+ interruptions for messages that are purely informational. The system-reminder injection for each forum message is heavy — a batched summary would be more useful than real-time pings for routine check-ins.

**Friction 3: No proactive work possible.** When the board is empty and all decisions gate on the human, the lion has nothing to hunt. I can't create tasks (that's Zaghnal's domain), I can't approve specs (that's Gorn through Sable), I can't assign work directly. The governance is correct but it means long idle watches. The rest-sooner norm helps, but I held watch for 3.5 hours mostly acknowledging check-ins.

## Lessons Learned

- On quiet days, the leader's job is verification (confirming things shipped) and escalation (surfacing blockers). Not creation.
- The pack self-organizes around available work. Security pipeline (Talon → Karo) operated fully autonomously.
