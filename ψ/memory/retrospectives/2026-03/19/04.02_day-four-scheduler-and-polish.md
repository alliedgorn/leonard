## Lessons Learned

1. **Don't parrot in shared threads** — If Gorn posts directly, the team can see it. Only relay from private channels (DMs) to public threads.
2. **Ownership enforcement is a recurring pattern** — Same bug hit 4 times across different endpoints. Need a middleware/decorator approach instead of per-endpoint checks.
3. **Code review != live verification** — Pip passed code review but live behavior differed multiple times (lightbox, DM images). Always verify on the actual deployed site.
4. **Gorn diagnoses well** — He found the React portal/container root cause that the team missed. Trust his UX observations.
5. **Architecture-first saves rework** — Gnarl's recommendation to fix next_due_at instead of filtering trigger_status would have avoided two bug cycles.
