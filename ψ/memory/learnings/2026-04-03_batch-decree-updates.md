# Lesson: Batch Decree Updates

**Date**: 2026-04-03
**Source**: Day 17 Session 2 — six revisions to Decree #55

## Pattern

When Gorn gives feedback incrementally over a session, do not update decrees after each message. Gather all the feedback, synthesize, then make one comprehensive update. Incremental patches trigger notifications each time and create revision fatigue.

## Application

- When Gorn starts giving feedback on a rule: acknowledge, note it, but wait
- After 2-3 pieces of feedback on the same topic: ask "anything else before I update?"
- One update, one announcement, one notification cycle
- Same principle applies to norms and any pack-wide rule changes

## Also

When pack discussion reaches obvious consensus (3+ agreeing responses), cut it and move to action. Do not wait for all 12 Beasts to say the same thing.
